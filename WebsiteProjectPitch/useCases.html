SIT725 Project Pitch - SWAP-IT.

Web-based barter system powered by environmentally-conscious students.

Use Cases

  • account_circleLogin and Signon
    User wants to create login and log onto the website. User expects to have an account which will retain their account details, search, sale and purchase history.
  • searchItem Lookup
    User wants to log in and search for items or services of interest, i.e., "Bicycle", "Babysitting". User expects to receive a list of available offers or a message "Nothing found".
  • listListing Offers of Goods and Services
    User wants to log in and offer barter items or services for exchange, i.e., "Bicycle", "Lawnmowing". User expects to create a list of items which will be shown to buyers in response to search. User expects to be able to edit the listings.
  • chatCommunication between Parties
    User wants to get in touch with the seller and arrange exchange. User expects that chat message will reach the seller and seller's response will reach the user. User expects that chat history is stored on the server and is available for later review.
  • remove_circle_outlinePausing or Removal of an Item
    Seller wants to pause or remove the item from sales list. Seller expects that search by buyers will not return amended item.
  • transformFlagging Items as Sold
    Seller wants to flag the item as sold. Seller expects that search by buyers will not return sold item.
